Write a Zumly driver

A Zumly transition driver is a single function: runTransition(spec, onComplete). The engine computes what moves where; your driver decides how. This page is the contract — the spec shape, the rules, the helpers, and a minimal driver you can copy and modify.

Get started → Full reference

When to write your own driver

Zumly already ships six built-in drivers — css (default), waapi, none, gsap, anime, and motion. Reach for a custom driver only when:

  • You want a transition shape the bundled drivers don't express — for example, a Lottie playback synced to the zoom progress, a WebGL ripple, or a parallax fade.
  • You're integrating with an animation engine that isn't on the built-in list (Theatre.js, Rive, Lottie, etc.) and you'd rather write 50 lines of glue than fork the package.
  • You want to wrap a built-in driver — for example, log every transition or compose two drivers — by importing the function and calling it from your wrapper.

Quick start

A driver is a function. Pass it to the constructor as transitions.driver: 

function myDriver(spec, onComplete) {
  // Animate based on spec.type, then call onComplete()
  onComplete()
}

const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  views: { home, detail },
  transitions: {
    driver: myDriver,
    duration: '500ms',
    ease: 'ease-in-out',
  },
})

await app.init()

That's it. Zumly wraps your function in a tiny adapter and calls it on every zoom-in, zoom-out, and lateral transition.

The spec object

Your function receives a single object with everything you need:

{
  type: 'zoomIn' | 'zoomOut' | 'lateral',
  currentView:  HTMLElement,         // incoming (zoom-in) or outgoing (zoom-out)
  previousView: HTMLElement,         // the parent
  lastView:     HTMLElement | null,  // grandparent (null at depth ≤ 2)
  currentStage: { views: [...] },    // pre-computed transforms (see below)
  duration:     '600ms',
  ease:         'ease-in-out',
  canvas:       HTMLElement,         // canvas container (zoom-out and lateral)
}

currentStage.views is an array indexed by role: views[0] is the current view, views[1] is the previous view, views[2] is the grandparent (when depth > 2). Each entry has: 

{
  viewName: 'detail',
  backwardState: {
    origin:    '125px 100px',
    transform: 'translate(50px, 60px) scale(0.25)',
    duration:  '500ms',
    ease:      'ease-in-out',
  },
  forwardState: {
    origin:    '125px 100px',
    transform: 'translate(100px, 50px)',
    duration:  '500ms',
    ease:      'ease-in-out',
  }
}

For zoom-in, animate from backwardState.transform to forwardState.transform. For zoom-out, animate the other direction.

The two rules

1. Always call onComplete() exactly once

The engine sets blockEvents = true before calling your driver and only resets it inside onComplete. If you forget, the UI freezes permanently. Use the createFinishGuard helper to make this impossible to get wrong: 

import {
  createFinishGuard,
  parseDurationMs,
  SAFETY_BUFFER_MS,
} from 'zumly/driver-helpers'

function runTransition(spec, onComplete) {
  const ms = parseDurationMs(spec.duration)

  const { finish } = createFinishGuard(() => {
    // cleanup work…
    onComplete()
  }, ms + SAFETY_BUFFER_MS)

  myAnimation.onfinish = finish
  // If the animation cancels, the safety timer calls finish() anyway.
}

2. Apply the final DOM state

When animation ends, the DOM must reflect the final state — the engine does NOT do this for you. Use the shared helpers: 

import {
  showViews,
  applyZoomInEndState,
  applyZoomOutPreviousState,
  applyZoomOutLastState,
  removeViewFromCanvas,
} from 'zumly/driver-helpers'

// Before animating — make views visible:
showViews(spec.currentView, spec.previousView, spec.lastView)

// After zoom-in completes:
applyZoomInEndState(currentView, currentStage)
applyZoomInEndState(previousView, currentStage)
if (lastView) applyZoomInEndState(lastView, currentStage)

// After zoom-out completes:
removeViewFromCanvas(currentView, canvas)
applyZoomOutPreviousState(previousView, currentStage.views[1].backwardState)
if (lastView) applyZoomOutLastState(lastView, currentStage.views[2].backwardState)

Available helpers

Imported from zumly/driver-helpers:

  • parseDurationMs(duration) — parse '1s'/'500ms'/number → ms
  • parseDurationSec(duration) — same but returns seconds
  • showViews(...elements) — remove the hide class and prime contentVisibility
  • applyZoomInEndState(el, stage) — final classes + transforms after zoom-in
  • applyZoomOutPreviousState(el, state) — final state for previous view after zoom-out
  • applyZoomOutLastState(el, state) — final state for grandparent after zoom-out
  • removeViewFromCanvas(el, canvas) — safe removal that handles wrapped elements
  • runLateralInstant(spec, onComplete) — instant lateral transition (no animation)
  • createFinishGuard(cleanup, timeoutMs) — once-only finish + safety timeout
  • SAFETY_BUFFER_MS — default safety buffer (150ms)

For drivers that interpolate computed matrices (when transform-origin varies between states):

  • readComputedMatrix(el, origin, transform) — read the browser-computed matrix (forces one reflow)
  • interpolateMatrix(from, to, t) — lerp between two matrix objects
  • matrixToString(m) & parseMatrixString(str) — round-trip conversion
  • identityMatrix() & lerp(a, b, t) — primitives

Minimal complete driver: opacity crossfade

Here's a complete custom driver — about 50 lines — that does an opacity crossfade instead of a zoom. Copy it as a starting point and replace the animation body with whatever effect you want. 

import {
  parseDurationMs,
  showViews,
  applyZoomInEndState,
  applyZoomOutPreviousState,
  applyZoomOutLastState,
  removeViewFromCanvas,
  runLateralInstant,
  createFinishGuard,
  SAFETY_BUFFER_MS,
} from 'zumly/driver-helpers'

export function runTransition(spec, onComplete) {
  const { type, currentView, previousView, lastView, currentStage, duration, canvas } = spec

  if (type === 'lateral') {
    runLateralInstant(spec, onComplete)
    return
  }

  const ms = parseDurationMs(duration)
  showViews(currentView, previousView, lastView)

  if (type === 'zoomIn') {
    currentView.style.opacity = '0'
    currentView.style.transition = `opacity ${ms}ms ease`
    previousView.style.transition = `opacity ${ms}ms ease`

    requestAnimationFrame(() => {
      currentView.style.opacity = '1'
      previousView.style.opacity = '0.3'
    })

    const { finish } = createFinishGuard(() => {
      currentView.style.transition = ''
      previousView.style.transition = ''
      previousView.style.opacity = ''
      applyZoomInEndState(currentView, currentStage)
      applyZoomInEndState(previousView, currentStage)
      if (lastView) applyZoomInEndState(lastView, currentStage)
      onComplete()
    }, ms + SAFETY_BUFFER_MS)

    currentView.addEventListener('transitionend', finish, { once: true })
    return
  }

  if (type === 'zoomOut') {
    currentView.style.transition = `opacity ${ms}ms ease`
    requestAnimationFrame(() => {
      currentView.style.opacity = '0'
      previousView.style.opacity = '1'
    })

    const { finish } = createFinishGuard(() => {
      currentView.style.transition = ''
      removeViewFromCanvas(currentView, canvas)
      applyZoomOutPreviousState(previousView, currentStage.views[1].backwardState)
      if (lastView) applyZoomOutLastState(lastView, currentStage.views[2].backwardState)
      onComplete()
    }, ms + SAFETY_BUFFER_MS)

    currentView.addEventListener('transitionend', finish, { once: true })
    return
  }

  onComplete()
}

Wrapping a built-in driver

You don't have to start from zero. Every built-in driver is exported by name; wrap one to add logging, analytics, or compose with another effect: 

import Zumly, { gsapTransition } from 'zumly'

function loggedDriver(spec, onComplete) {
  console.log('zumly transition', spec.type, spec.currentStage.views[0].viewName)
  gsapTransition(spec, onComplete)
}

const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  views: { home, detail },
  transitions: { driver: loggedDriver, duration: '500ms' },
})

Testing your driver

The 'none' driver is the canonical reference: it applies the final state instantly and calls onComplete() synchronously. Your driver should produce the same final DOM state, just animated. Key invariants to verify:

  • After zoom-in, .is-current-view exists with the correct dataset.viewName.
  • After zoom-out, the previous view is now .is-current-view and the old current is removed from the DOM.
  • onComplete() is always called, even if elements are removed mid-animation.
  • The engine resets blockEvents in its own onComplete wrapper, but only if you call it.
const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  views: { home, detail },
  transitions: { driver: myDriver, duration: '100ms' },
})
await app.init()
await app.zoomTo('detail')
expect(app.getCurrentViewName()).toBe('detail')
app.back()
expect(app.getCurrentViewName()).toBe('home')
