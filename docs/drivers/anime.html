Driver · Anime.js

Zumly + Anime.js

Run Zumly's zoom transitions through Anime.js. The bundled adapter interpolates the browser-computed CSS matrix between view states, so origins, scales, and translations stay aligned even when transform-origin shifts during the animation.

Get started → View on GitHub

Why use the Anime.js driver?

The Anime.js adapter is a good choice when you need geometrically faithful intermediate frames. Instead of tweening transform as a string, the driver reads the computed matrix at the start and end of the transition and interpolates between them frame-by-frame.

That matters when:

  • The current and previous views have different transform-origin values (very common in zoom-into-element flows where each child has its own bounding box).
  • You want a smooth visual line between two states that browsers would otherwise stitch with a slight pop because they tween origin and transform separately.
  • You're already using Anime.js elsewhere and want the same easing engine across your UI for visual consistency.

Install

The driver ships inside the zumly package. Add Anime.js as a global:

npm install zumly
<script src="https://cdnjs.cloudflare.com/ajax/libs/animejs/3.2.2/anime.min.js"></script>
Anime.js v3. The bundled adapter targets Anime.js v3 (the global anime function). v4's named exports work too, as long as you assign the default export to globalThis.anime before Zumly initialises.

Quick start

import Zumly from 'zumly'
import 'zumly/css'

const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  views: { home, gallery, photo },
  transitions: {
    driver: 'anime',
    duration: '500ms',
    ease: 'easeInOutQuad',
  },
})

await app.init()

Any Anime.js easing string is valid for the ease field — the adapter passes it straight through to anime(). CSS-style strings like 'ease-in-out' are also accepted and mapped to a sensible default curve.

Working example: photo gallery drill-down

const app = new Zumly({
  mount: '.canvas',
  initialView: 'gallery',
  views: { gallery, photo },
  transitions: {
    driver: 'anime',
    duration: '700ms',
    ease: 'easeOutExpo',
    effects: ['blur(3px) brightness(0.8)', 'blur(8px) brightness(0.4)'],
  },
})

await app.init()

Because the driver interpolates computed matrices, each photo zooms into its exact bounding rectangle without the typical "pop" you get when transform-origin changes late in the animation.

What the adapter actually does

  • Calls showViews() on current, previous, and grandparent views.
  • Uses readComputedMatrix() to capture the start and end matrix for each view (this forces a single layout read — see the API reference for the timing).
  • Drives Anime.js with a { progress } object and applies matrixToString(interpolateMatrix(from, to, t)) to each view's transform on every tick.
  • On complete, applies the canonical end state via applyZoomInEndState() / applyZoomOutPreviousState() and calls onComplete().

Lateral transitions are passed to runLateralInstant() — Zumly's lateral slide is geometric, not curved, so adding Anime.js to it adds no value.

Direct import (advanced)

import Zumly, { animeTransition } from 'zumly'

const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  views: { home, detail },
  transitions: {
    driver: animeTransition,
    duration: '500ms',
  },
})

Wrapping or extending the function works the same way as for any built-in driver. See writing a custom driver for the full spec shape.

Common gotchas

  • Anime.js must be on globalThis before Zumly initialises. The adapter checks at construction time. If Anime.js is missing, the driver warns once and falls through to onComplete() — views still navigate, just instantly.
  • Matrix interpolation forces one extra layout read per transition. On modern devices this is invisible; on very low-end mobile, the gsap or css drivers (which animate the transform string directly) are slightly cheaper.
  • Keep custom transforms off the views during a zoom. The driver writes a full matrix(...) on every tick; concurrent inline transforms will be overwritten.
