API Reference

Complete reference for every option, method, and event.

Constructor Options

Required and optional parameters passed to new Zumly({...}).

OptionTypeRequiredDefaultDescription
mountstringYesCSS selector for the canvas element (must have zumly-canvas class)
initialViewstringYesName of the first view to display
viewsobjectYesMap of view names to view sources
preloadstring[][]Views to resolve and cache eagerly during init()
deferredbooleanfalseDefer content rendering until after animation completes
debugbooleanfalseEnable debug logging to console
componentContextMap | objectnew Map()Shared context passed to function/object view sources
deferred boolean default: false
When enabled, view content is inserted after the zoom animation completes instead of before. Useful for heavy views that might cause jank during animation. Per-trigger override via data-deferred attribute.
preload string[] default: []
Resolves and caches listed views during init(). Improves perceived performance for views with async sources or URL fetches. Compare with and without preload to see the loading difference.

Transitions

Configure how zoom animations look and feel. All nested under transitions: {...}.

driver string | function default: 'css'
The animation engine used for zoom transitions. Built-in: 'css', 'waapi', 'none'. External (require library): 'anime', 'gsap', 'motion'. Or pass a custom (spec, onComplete) => void function.
duration string default: '1s'
CSS duration string for animations ('500ms', '1.5s'). Per-trigger override via data-with-duration.
ease string default: 'ease-in-out'
CSS easing function. Any valid CSS value: 'ease', 'linear', 'cubic-bezier(...)'. Per-trigger override via data-with-ease.
cover 'width' | 'height' default: 'width'
Which dimension of the trigger element determines the zoom scale. 'width' scales based on trigger width, 'height' scales based on trigger height. Most visible with non-square triggers.
effects [string, string] default: ['none', 'none']
CSS filter strings applied to background views during zoom. First value for the previous view, second for the last (deepest background) view. Example: ['blur(3px) brightness(0.7)', 'blur(8px) saturate(0)'].
stagger number default: 0
Progressive delay (ms) between view layers during transitions. Current view starts immediately, previous after 1×stagger, last after 2×stagger. Creates a cascading animation effect.
hideTrigger boolean | 'fade' default: false
How to hide the clicked trigger element during zoom-in. false = visible, true = visibility:hidden, 'fade' = smooth opacity crossfade.

Navigation Options

Control lateral, depth, and input navigation behavior.

lateralNav

OptionTypeDefaultDescription
arrowsbooleantrueShow prev/next arrow buttons
dotsbooleantrueShow dot indicators for siblings
keepAliveboolean | 'visible'falseKeep views in DOM: false = remove, true = hidden, 'visible' = visible
lateralNav boolean | object default: true
Enable same-level navigation between sibling views using goTo(name, { mode: 'lateral' }). Parent view shifts spatially to track which child is active.
keepAlive boolean | 'visible' default: false
Controls whether lateral views are preserved in the DOM after navigating away. true keeps them hidden (display:none), 'visible' keeps them visible in the background. Use the counter to see state preservation.

depthNav

OptionTypeDefaultDescription
buttonbooleantrueShow zoom-out back button
indicatorbooleantrueShow depth level dots
depthNav boolean | object default: true
Controls the depth navigation UI: a back button and depth indicator dots. Pass false to disable all depth UI, or an object to selectively toggle.

inputs

OptionTypeDefaultDescription
clickbooleantrueEnable click/tap zoom-in on .zoom-me elements
keyboardbooleantrueArrow keys for lateral, up for zoom-out
wheelbooleantrueMouse wheel zoom-out
touchbooleantrueTouch zoom-in and swipe lateral
inputs boolean | object default: true
Selectively enable or disable input methods. Pass false to disable all, or an object to toggle individual inputs. Useful for kiosk mode or custom input handling.

Methods

Instance methods for programmatic control.

MethodReturnsDescription
init()PromiseInitialize and render the initial view
zoomTo(name, opts?)PromiseProgrammatic zoom into a named view
goTo(name, opts?)PromiseNavigate: { mode: 'depth' | 'lateral' }
back()Promise | voidPop lateral history first, then zoom out
zoomIn(el)PromiseZoom into the view indicated by a trigger element
zoomOut()voidZoom out one level. No-op at root
zoomLevel()numberCurrent zoom depth (0 = root)
getCurrentViewName()string | nullName of the currently active view
on(event, fn)thisSubscribe to lifecycle event (chainable)
off(event, fn?)thisUnsubscribe. Omit fn to remove all for that event
destroy()voidClean up all listeners and state
zoomTo() / goTo() / back() Promise
Navigate programmatically without user clicks. zoomTo() for depth navigation, goTo() with { mode: 'lateral' } for same-level, back() to go back.
// Programmatic navigation
const app = new Zumly({ mount: '#canvas', initialView: 'home', views });
await app.init();

await app.zoomTo('detail');         // zoom in
await app.zoomTo('deep');           // zoom deeper
app.back();                         // zoom out

await app.goTo('page2', { mode: 'lateral' });  // slide to sibling
app.back();                         // back to previous sibling

console.log(app.zoomLevel());           // 2
console.log(app.getCurrentViewName());  // 'detail'

View Sources

Views can be defined in many ways. The resolver detects the type automatically.

TypeExampleCached
HTML string'<div class="z-view">...</div>'Indefinitely
URL'/views/detail.html'5 min TTL
Function(ctx) => '<div>..</div>'No
Async functionasync () => await fetch(...)No
Object{ render(ctx) { return '...' } }No
HTMLElementdocument.createElement('div')Cloned
Web component'my-component'No
View Sources Multiple formats
Each card zooms into a view created from a different source type. Compare HTML strings, functions, objects with render(), and async sources.
// HTML string
const myView = '<div class="z-view">Hello</div>';

// Function (called fresh each time)
function myView({ target, props }) {
  return `<div class="z-view">${Date.now()}</div>`;
}

// Async function
async function myView() {
  const html = await fetch('/partial.html').then(r => r.text());
  return html;
}

// Object with render() and optional mounted()
const myView = {
  render(ctx) { return '<div class="z-view">...</div>'; },
  mounted() { console.log('inserted!'); }
};

// URL (fetched and cached for 5 minutes)
const myView = '/views/detail.html';

Per-Trigger Attributes

Override global settings on individual trigger elements using data attributes.

AttributeDescription
data-toRequired. Target view name to zoom into
data-with-durationOverride transition duration (e.g. "2s")
data-with-easeOverride easing function
data-with-effectsOverride effects (pipe-separated: "blur(5px)|blur(10px)")
data-deferredOverride global deferred setting for this trigger
data-*Any data attribute becomes a prop in ViewContext.props
 
<div class="zoom-me"
     data-to="detail"
     data-with-duration="2s"
     data-with-ease="cubic-bezier(.4,0,.2,1)"
     data-with-effects="blur(5px)|blur(10px)"
     data-deferred
     data-id="42"
     data-category="settings">
  Zoom in
</div>

// In the target view function:
function detail({ props }) {
  console.log(props.id);       // "42"
  console.log(props.category); // "settings"
}
