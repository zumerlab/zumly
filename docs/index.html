Build hierarchical, zoom-driven navigation for the web. Spatial transitions that preserve context, powered by pluggable animation drivers.

Why Zumly

Everything you need for zoom-based navigation, nothing you don't.

❖

Spatial Continuity

Users zoom into elements to explore deeper content. Context is preserved through smooth hierarchical transitions.

⚙

Pluggable Drivers

CSS, WAAPI, GSAP, Anime.js, Motion, or your own. Swap without touching app code.

▩

Multiple View Sources

HTML strings, URLs, async functions, render objects, HTMLElements, web components.

⚡

Smart Prefetching

Eager preload, hover prefetch, scan prefetch. Views are ready before the user clicks.

↔

Lateral Navigation

Same-depth view swaps with spatial slide transitions. Navigate sideways at any zoom level.

⚪

Zero Dependencies

Core has no deps. Animation libraries are optional, loaded only if you use their driver.

Hello World

The simplest Zumly app: two views, click to zoom in, click outside to zoom out.

Live 
const hello = `
  <div class="z-view" style="width:500px; height:250px">
    <h3>Hello World</h3>
    <span class="zoom-me" data-to="world">Zoom</span>
  </div>`;

const world = `
  <div class="z-view" style="width:400px; height:200px">
    <h3>World</h3>
    <p>Click outside or press Escape to zoom out</p>
  </div>`;

new Zumly({
  mount: '#my-canvas',
  initialView: 'hello',
  views: { hello, world },
}).init();

Deep Zoom

Navigate multiple levels deep. Each zoom preserves the spatial context of parent views with blur and parallax.

3 levels · effects · parallax 
new Zumly({
  mount: '#canvas',
  initialView: 'continent',
  views: { continent, country, city },
  transitions: {
    duration: '600ms',
    cover: 'height',
    effects: ['blur(3px) brightness(0.7)', 'blur(8px) brightness(0.4)'],
    parallax: 0.3,
    hideTrigger: 'fade',
  },
}).init();

Lateral Navigation

Swap views at the same zoom level with slide transitions. The parent view shifts spatially to track which child is active. Use goTo(name, { mode: 'lateral' }) to navigate and back() to return through the lateral history.

Same-level navigation
keepAlive 
// Parent view has triggers pointing to each sibling:
const menu = `
  <div class="z-view" style="width:600px; height:400px">
    <div class="zoom-me" data-to="page1">Page 1</div>
    <div class="zoom-me" data-to="page2">Page 2</div>
    <div class="zoom-me" data-to="page3">Page 3</div>
  </div>`;

// Zoom into page1 first, then navigate laterally:
app.goTo('page2', { mode: 'lateral' });
app.goTo('page3', { mode: 'lateral' });

// back() pops lateral history, then zooms out
app.back(); // → page2
app.back(); // → page1
app.back(); // → menu (zoom out)

// keepAlive: views stay in DOM when navigating laterally
lateralNav: { keepAlive: true }

View Sources

Views can be defined in many ways. The resolver detects the type automatically.

All view types 
// 1. HTML string
const myView = '<div class="z-view">Hello</div>';

// 2. Function → string or HTMLElement
function myView({ target }) {
  return `<div class="z-view">${Date.now()}</div>`;
}

// 3. Async function (fetch, API call, etc.)
async function myView() {
  const html = await fetch('/partial.html').then(r => r.text());
  return html;
}

// 4. Object with render() and optional mounted()
const myView = {
  render(ctx) { return '<div class="z-view">...</div>'; },
  mounted() { console.log('inserted!'); }
};

// 5. HTMLElement (cloned each time)
const el = document.createElement('div');

// 6. URL (fetched with 5-min cache)
const myView = '/views/detail.html';
