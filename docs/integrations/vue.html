Zumly with Vue

Add zoom-based navigation to any Vue 3 application. Zumly is a framework-agnostic Zooming User Interface (ZUI) engine — your Vue components mount into zoomable views with no plugin, no Vue-specific bridge, and no Zumly-aware code in your Single File Components.

Get started → View on GitHub

Why use Zumly with Vue?

Vue handles your component tree; Zumly handles the spatial relationship between trees. Each Zumly view is a self-contained DOM subtree where you can call createApp(...).mount(target) exactly the same way you'd bootstrap Vue into any other container. Zumly takes care of the zoom transition, the focus origin, and the navigation stack.

That means Zumly doesn't replace Vue Router or compete with Pinia. It's an orthogonal layer for spatial, focus-driven navigation: instead of swapping URLs in a flat space, users zoom into a card, a row, an item, and the Vue component for that view renders inside the zoomed-in target.

Common use cases for Zumly + Vue:

  • Dashboards with hierarchical drill-down (KPIs → charts → raw data)
  • IoT and home control panels (rooms → devices → settings)
  • Settings UIs that zoom into nested sections instead of pushing routes
  • Kiosk and signage interfaces where focus is the primary navigation metaphor
  • Product configurators with zoomable detail levels

Install

npm install zumly

Zumly has zero dependencies. The package ships ESM, CJS, and a UMD build for plain HTML.

Quick start

The simplest pattern: declare your views as function views. Zumly hands you a fresh target element, and you mount a Vue app into it with createApp. 

import Zumly from 'zumly'
import 'zumly/css'
import { createApp } from 'vue'
import Dashboard from './Dashboard.vue'

const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  views: {
    home: '<div class="z-view"><div class="zoom-me" data-to="dashboard" data-id="42">Open dashboard</div></div>',
    dashboard: ({ target, props }) => {
      createApp(Dashboard, { id: props.id }).mount(target)
    }
  }
})

await app.init()

Any element inside a view with class zoom-me and a data-to attribute becomes a zoom trigger. Data attributes on the trigger (like data-id="42") get passed into your Vue component as props.id.

Working example: Dashboard with drill-down

A two-level pattern where a list of items zooms into a detail view, with the Vue component receiving the selected item's id from the trigger.

App.vue

<script setup>
import { onMounted, onBeforeUnmount, ref } from 'vue'
import { createApp } from 'vue'
import Zumly from 'zumly'
import 'zumly/css'

import ItemList from './ItemList.vue'
import ItemDetail from './ItemDetail.vue'

const canvas = ref(null)
let zumly

onMounted(async () => {
  zumly = new Zumly({
    mount: canvas.value,
    initialView: 'list',
    views: {
      list: ({ target }) => {
        createApp(ItemList).mount(target)
      },
      detail: ({ target, props }) => {
        createApp(ItemDetail, { id: props.id }).mount(target)
      }
    },
    transitions: {
      duration: '600ms',
      ease: 'cubic-bezier(.4,0,.2,1)',
      effects: ['blur(4px) brightness(0.6)', 'blur(10px) brightness(0.3)']
    }
  })
  await zumly.init()
})

onBeforeUnmount(() => zumly?.destroy())
</script>

<template>
  <div ref="canvas" class="zumly-canvas" style="height: 100vh"></div>
</template>

ItemList.vue

<script setup>
const items = [
  { id: '1', name: 'Engine telemetry' },
  { id: '2', name: 'Battery health' },
  { id: '3', name: 'Brake system' }
]
</script>

<template>
  <div class="z-view">
    <h1>Subsystems</h1>
    <div
      v-for="item in items"
      :key="item.id"
      class="zoom-me"
      data-to="detail"
      :data-id="item.id">
      {{ item.name }}
    </div>
  </div>
</template>

ItemDetail.vue

<script setup>
defineProps(['id'])
</script>

<template>
  <div class="z-view">
    <h1>Detail for {{ id }}</h1>
  </div>
</template>

Lifecycle and cleanup

Function views in Zumly are never cached — they resolve fresh each time the user zooms in. That's the right behavior for Vue: every zoom-in creates a new Vue application instance, every zoom-out unmounts it, and Vue's normal lifecycle hooks (onMounted, onBeforeUnmount, etc.) run as expected.

Tip: If you need to keep a component instance alive across navigation (for example, a chart that shouldn't re-fetch its data), use an object view with render() and mounted() methods, and store your Vue app instance in a closure variable instead of recreating it.

Sharing context (Pinia, Vue Router, plugins)

Pass your Pinia instance, router, or any Vue plugin to every view via componentContext on the Zumly constructor. Zumly forwards it as context to all function and object views, so each Vue app can install the same plugins on bootstrap. 

import { createPinia } from 'pinia'

const pinia = createPinia()

const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  componentContext: { pinia },
  views: {
    dashboard: ({ target, context }) => {
      createApp(Dashboard)
        .use(context.pinia)
        .mount(target)
    }
  }
})

TypeScript

Zumly ships TypeScript types out of the box. View functions are typed with the target, props, and context parameters, so your Vue mount code gets full IntelliSense. 

import Zumly, { ViewFunction } from 'zumly'
import { createApp } from 'vue'
import Dashboard from './Dashboard.vue'

const dashboard: ViewFunction = ({ target, props }) => {
  createApp(Dashboard, { id: props.id as string }).mount(target)
}

Common gotchas

  • The target is a fresh <div> created by Zumly — don't try to mount into document.body or query for an existing element. Just call createApp(...).mount(target).
  • Don't forget the z-view class on your Single File Component's root element. Zumly auto-adds it if missing, but being explicit avoids surprises with CSS selectors.
  • Trigger data attributes become strings. data-id="42" arrives as props.id === "42", not as a number. Cast or parse on the Vue side.
  • Each view gets its own createApp instance. If you depend on a single global Pinia store or Vue Router, install them on every view via componentContext rather than relying on a top-level app.
