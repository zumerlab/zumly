Zumly + Orbit CSS

Orbit CSS is a polar (radial) layout system from Zumerlab — siblings of Zumly. Together they're a natural fit: Orbit gives you radial dashboards where every orbit is a logical ring of children, and Zumly lets the user zoom into any of those children to make it the new focus.

Get started → Orbit CSS on GitHub

Why combine them?

Most CSS layout systems are rectangular: grids, flex rows, columns. Orbit CSS introduces a different primitive — the orbit, a circular ring around a centre point, with children placed at angles around it. That maps naturally onto radial dashboards, round control panels, dial-style instruments, watch-face UIs, and any interface where the natural metaphor is "around a centre" rather than "in a row."

A focus-driven ZUI on top of a radial layout is doubly powerful:

  • The centre of the orbit is the obvious focus — it's where the eye lands. Use it for the current view.
  • The orbit ring contains siblings — perfect for lateral navigation.
  • The outer orbits are children of the centre — perfect for depth triggers. Click one and Zumly zooms into it as the new centre.

The result is a UI that feels physical: the user is navigating a structure that looks and behaves like a dial, with focus always at the middle and context always at the rim.

Install

Install both packages — they're zero-dependency and work alongside each other.

npm install zumly @zumerlab/orbit

Or load them from a CDN as plain CSS / UMD scripts:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@zumerlab/orbit/dist/orbit.min.css">
<script src="https://cdn.jsdelivr.net/npm/zumly/dist/zumly.umd.js"></script>

Quick start

Build a radial home view with Orbit's .o-orbit primitive, then make each ring item a Zumly trigger by adding class="zoom-me" and data-to. 

import Zumly from 'zumly'
import 'zumly/css'
import '@zumerlab/orbit/dist/orbit.min.css'

const home = `
  <div class="z-view o-orbit">
    <div class="o-center">Home</div>
    <div class="o-ring" data-radius="180">
      <div class="o-item zoom-me" data-to="livingRoom">Living room</div>
      <div class="o-item zoom-me" data-to="kitchen">Kitchen</div>
      <div class="o-item zoom-me" data-to="bedroom">Bedroom</div>
      <div class="o-item zoom-me" data-to="garage">Garage</div>
    </div>
  </div>
`

const livingRoom = `
  <div class="z-view o-orbit">
    <div class="o-center">Living room</div>
    <div class="o-ring" data-radius="160">
      <div class="o-item zoom-me" data-to="lights">Lights</div>
      <div class="o-item zoom-me" data-to="thermostat">Thermostat</div>
      <div class="o-item zoom-me" data-to="speakers">Speakers</div>
    </div>
  </div>
`

const app = new Zumly({
  mount: '.canvas',
  initialView: 'home',
  views: { home, livingRoom, lights, thermostat, speakers, kitchen, bedroom, garage },
  transitions: { driver: 'css', duration: '600ms', ease: 'ease-in-out' }
})

await app.init()

Each .o-item is a regular DOM element — Orbit positions it on the ring, Zumly turns it into a zoom trigger. There's no glue layer between the two libraries.

Working example: see the showcases

Two of the Zumly showcases are real Zumly + Orbit CSS apps:

  • Home Assistant — a radial home-control dashboard. Each room is a ring item; zooming into a room reveals its devices on a smaller ring; zooming into a device reveals its controls at the centre.
  • Mission Control — a radial ops centre with concentric orbits for stations, telemetry feeds, and individual sensors. Each ring is a depth level; zoom in to focus, zoom out to overview.

Both showcases run on the live demo and are open-source — read the source to see exactly how the markup and the constructor options fit together.

Design tips for Zumly + Orbit

  • Match orbit depth to zoom depth. If the user can see two orbit rings in the parent view, they should expect that zooming into a ring item reveals its own ring at the next depth. Consistency builds the spatial intuition.
  • Keep the centre as the focus. The centre of an orbit is the visual anchor — always put the current view's identity (label, icon, primary control) there.
  • Use lateral for orbit siblings. If the user is focused on "Lights" and wants to switch to "Thermostat" without going up to the room view, use lateral navigation rather than two depth animations.
  • Respect prefers-reduced-motion. Both libraries respect the OS-level setting; for radial UIs, the rotation cues can be disorienting at full intensity, so consider a more conservative duration and ease for those users.

Why these two libraries fit

Zumly and Orbit CSS share a design philosophy: they're both about spatial primitives rather than visual chrome. Orbit doesn't paint anything — it just positions elements on a circle. Zumly doesn't render anything — it just animates between views. You bring the content; the libraries provide the geometry.

That means you can use Zumly + Orbit with React, Vue, Svelte, Angular, or plain HTML. Each view is just a DOM subtree; whether you write it as a string template, a React component, or a Vue SFC is your call. The two libraries don't care.
